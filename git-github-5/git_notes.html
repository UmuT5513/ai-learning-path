Git Temel ve Orta Seviye Notları

1. Git Kurulumu ve Konfigürasyon

Windows: https://git-scm.com/download/win adresinden .exe dosyasını indirip kur. MacOS: Terminal'de git --version yaz, yoksa yükleme talimatı çıkar. Linux (Ubuntu): sudo apt update && sudo apt install git Kurulumdan sonra: git config --global user.name "Senin İsmin" git config --global user.email "mail@ornek.com"

2. Git Nedir ve Neden Kullanılır?

Git bir versiyon kontrol sistemidir. Kod değişikliklerini kayıt altına alır, proje geçmişini yönetir, hatalı değişiklikleri geri almayı ve ekip çalışmasını kolaylaştırır.

3. Terminal/Command Prompt Temelleri

cd klasör_adı # Klasöre gitmek ls / dir # Dosya ve klasörleri görmek pwd # Bulunduğun konumu görmek mkdir yeni_klasor # Klasör oluşturmak

4. Basit Git Projesi Başlatmak

mkdir ilk-git-projem cd ilk-git-projem git init echo "Bu benim ilk git projem." > readme.txt git status git add readme.txt git commit -m "İlk dosyamı ekledim" git log

5. Branch (Dal) Kullanımı

git branch # Dalları listele git branch ozellik1 # Yeni dal oluştur git checkout ozellik1 # Dalı değiştir git add . git commit -m "Dalda değişiklik" git checkout main git merge ozellik1 # Dalı ana dala birleştir

6. Push/Pull ve GitHub Kullanımı

GitHub'da yeni repo oluştur. git remote add origin https://github.com/KULLANICIADIN/REPOADI.git git push -u origin main # İlk push git pull origin main # Uzak depodan çekmek

7. Commit, Undo, Reset, Revert

Henüz commit edilmemiş dosyayı eski haline döndürmek: git checkout -- dosya.txt Staged dosyayı geri almak: git reset HEAD dosya.txt Son commit'i silmek: git reset --soft HEAD~1 (veya --hard ile tamamen) Push'lu commit'i geri almak: git revert <commit_kodu>

8. Conflict (Çakışma) ve Çözümü

İki dalda aynı satırda değişiklik yapınca merge'de conflict oluşur. Çakışan dosyada: <<<<<<< HEAD senin değişikliğin ======= diğer branch'teki değişiklik >>>>>>> yedek Çözüm: Satırları düzenle, işaretleri sil. git add dosya.txt ve git commit -m "Conflict çözüldü"

9. .gitignore Kullanımı

Proje kökünde .gitignore dosyası oluştur. .gitignore'a eklemek için: *.log .env node_modules/ .gitignore sadece eklenmemiş dosyalar için çalışır.

10. .env Dosyası ve Güvenliği

.env dosyası içinde gizli bilgiler saklanır. .gitignore dosyasına .env satırı ekle: .env Daha önce eklediysen: git rm --cached .env && git commit -m ".env dosyası ignorelandı" Başka geliştiriciler için .env.example dosyası bırakılabilir.
